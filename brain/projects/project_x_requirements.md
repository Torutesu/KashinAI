# Project X — 要件確認

## 要件確認の目的

PoC提案（project_x_proposal.md）で合意した方向性をもとに、PoC実施前に確定させるべき要件を整理する。

## データ提供に関する要件

- 提供予定データ: 案件属性情報（業種、職種カテゴリ、求人条件のレンジ、対応エリア等）
- 除外データ: 求職者個人の氏名・連絡先・経歴等の個人情報は提供対象から除外
- データ形式: 現状はスプレッドシート運用。CSVまたはスプレッドシート共有での提供を想定
- 提供頻度: PoC期間中は週次または隔週での更新を想定（要調整）

## 対象業務の確定範囲

- 確定: 営業資料作成の自動化（フェーズ1）
- 保留: 問い合わせ対応の効率化（フェーズ2、次PoC候補として継続検討）

## 体制に関する要件

- Northlight側の実務担当者2〜3名がPoC期間中のフィードバックに対応
- 週次の短時間フィードバックMTG（30分程度）を想定
- 北原氏が最終的な意思決定・社内調整の窓口

## セキュリティ・コンプライアンス要件

- 人材紹介業として個人情報保護に関する社内規程があり、個人情報を含むデータの外部提供には追加の確認プロセスが必要
- 現時点では個人情報を含まないデータのみでPoCを開始する方針のため、追加確認プロセスは不要と見込まれるが、最終確認は次回MTGで行う

## 未確定事項（次回MTGで確認）

1. データ提供の正式な頻度・方法の確定
2. 費用対効果の具体的な試算値の提示と合意
3. セキュリティに関する社内稟議の要否（情報システム部門確認の要否）
4. PoC契約書の内容確認・締結スケジュール
