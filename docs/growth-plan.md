> 作成: 2026-07-15。プロダクト監査（Issue #10 / PR #11 完了時点）を前提とした成長計画。
> 実装ハンドオフ先: Claude Opus 4.8（Issue 単位で受け入れ条件付き）。

## 0. 目標の物差し

史上最速級の実績: Lovable が 8ヶ月で $100M ARR、Cursor がローンチ約21ヶ月で $100M ARR。
「6ヶ月で $100M」は史上最速記録更新であり計画値にならない。本計画の目標は:

- **M1**: 一般配布ローンチ、課金開始
- **M3**: $1M ARR ランレート（課金 約4,200人 @ $20/月）
- **M6**: $10M ARR ランレート（課金 約42,000人 + チーム課金）— それ自体トップYC級
- **$100M への継続条件**: チーム/エンタープライズ展開 + Windows + 英語圏（12〜24ヶ月）

### ARR 算術

| 目標 | 必要MRR | $20/月の課金者数 | DL→課金5%換算の必要インストール |
|---|---|---|---|
| $1M ARR | $83k | 4,200 | 〜84,000 |
| $10M ARR | $833k | 42,000 | 〜840,000 |
| $100M ARR | $8.3M | 417,000 | 〜8,300,000 |

→ 個人×macOSのみ×日本語では $100M に構造的に届かない。
**個人PLG → チーム課金 → Windows → 英語圏** の4段が必須。

## 1. 現状診断（コード実測ベース、2026-07-15）

### 資産
- 文脈キャプチャエンジン（AX→ブラウザ→Chromiumセッション→OCRの多段フォールバック）— 競合優位の核
- fixture 回帰基盤（509 unit tests、ライブ失敗→回帰化の運用設計）
- SNS/コード画面で会社メモを混ぜない文脈判別（live-context）
- 出荷レイヤー: CI、署名/notarize 設定、自動更新、オンボーディング（PR #11 で整備済み）

### 構造的ボトルネック TOP5

1. **売上回収の仕組みがゼロ**: BYOK（ユーザー自身のAPIキー）。アカウント/課金/ライセンス/バックエンドなし。
   現状は何人使っても ARR = 0。BYOK は同時に最大のアクティベーション障壁でもある。
2. **計測ゼロ**: テレメトリ/クラッシュレポート/ファネルが一切ない（posthog/sentry等の依存なし）。
3. **レイテンシ構造が予算オーバー**: 実タイムアウト積算で最悪30秒超
   （osascript 5s / AXヘルパー 2.5s / screencapture 8s / OCR 10s / GBrain 10s / LLM 非ストリーミング同期 fetch）。
   `formatCurrentContext` が全文脈を無制限にプロンプト投入（遅くて高コスト）。ストリーミングなし。
   目標: **tap→ペースト P50 ≦ 2.5s、初トークン P50 < 1.5s**。
4. **日本語ハードコード**: `SYSTEM_PROMPT` が日本語固定。出力言語の自動判定なし。英語圏が閉鎖。
5. **配布未完**: アイコンなし、LP/ドキュメント/プライバシーポリシー/ToS なし、実署名ビルド未検証。
   ※ Mac App Store はサンドボックスで AX キャプチャが成立しないため**直接配布一択**（現方針は正しい）。

### 競合ポジション（2026-07）
- Cluely（$20/月〜）: 会議/面接カンニング寄り。2025年に8.3万ユーザーの情報漏洩 → 「ローカルファースト+透明性」が差別化の追い風。
- Granola（無料 / Business $14/席 / Enterprise $35/席）: 会議メモ特化。価格の隣接ベンチマーク。
- Microsoft Recall / Claude Cowork / Perplexity 常駐エージェント: OS・エージェント層。
- **「今見ている画面に合う文章を1キーで貼る × 会社メモ」の書き味特化ポジションは空いている。**

## 2. 戦略方針（4つの転換）

1. **BYOK → ホステッド推論**: 薄いプロキシ（認証・従量メーター・モデルルーティング・SSE中継）。
   アプリはログインだけで即使える。BYOK はパワーユーザー/エンタープライズ向けオプションとして残す。
2. **価格**: Free（1日N回）/ Pro **$20/月** / Team **$30/席/月**。
3. **Wedge = Option タップの魔法**（獲得・デモ動画映え）、**リテンション = 会社メモ**（解約防止・チーム展開）。
4. **EN-first を最初から**: プロンプト・UI・LP を英日両対応で Phase 1 からグローバル配布。

## 3. フェーズ別実装計画

### Phase 0「売れる状態にする」（Week 1–3）
| # | 項目 | 受け入れ条件 |
|---|---|---|
| 0-1 | 計測基盤（PostHog + Sentry、匿名・オプトアウト付き） | install→onboarding→初生成→初ペーストのファネルがダッシュボードで見える |
| 0-2 | レイテンシ計測（captureTrace にステージ別 ms 計時） | P50/P95 がステージ別に取得できる |
| 0-3 | バックエンドMVP（Hono on CF Workers/Fly: JWT認証 + 推論プロキシ + 使用量メーター） | アプリがAPIキーなしで生成できる |
| 0-4 | アカウント + Stripe 課金 + entitlement（Free回数制限→ペイウォール） | カード課金→Pro解放が通る |
| 0-5 | ストリーミング生成（SSE、パネル逐次表示、tap は完了時ペースト） | 初トークン P50 < 1.5s |
| 0-6 | 配布完成（アイコン、実署名/notarize検証、LP、プライバシーポリシー/ToS） | クリーンMacで DL→起動→初ペーストが通る |
| 0-7 | プロンプト i18n（EN/JA、出力言語自動判定） | 英語画面で英語出力、日本語画面で日本語出力 |

実装順の推奨: **0-1 → 0-6 → 0-3/0-4 → 0-5**（計測なしで他を進めると効果検証ができない）。

### Phase 1「アクティベーションとリテンション」（Week 3–8）
- レイテンシ最適化: キャプチャ並列化+早期打ち切り、コンテキストのトークン予算化、tap経路は高速小型モデル、HTTP keep-alive。目標 tap→ペースト P50 ≦ 2.5s
- Aha-moment: オンボーディング最終stepで「その場でデモ生成→ペースト体験」。install→初ペースト中央値 < 5分
- フィードバック: 生成に👍👎、ペースト後編集率（ローカル計算・プライバシー安全）を品質指標化
- LLM 評価ハーネス（§4）: プロンプト変更の回帰ゲート
- リファラル: 招待で両者に Pro 1ヶ月
- 週次コホートダッシュボード: D1/D7/D30、activation率、tap/日、転換率

### Phase 2「チーム課金と拡散」（Month 2–4）
- 共有チームメモリ（brain/ のホステッド化: サーバー側 GBrain + 同期、招待リンク = バイラルループ）
- Team 課金（席数・管理画面・メンバー管理）
- Slack/Gmail 連携（文脈を深くする = 解約防止）、テンプレートギャラリー
- **Windows 移植着手**（UIAutomation + Windows OCR でキャプチャ層を再実装。TAM最大の拡張、工数最大なので早期着手）

### Phase 3「スケールとエンタープライズ」（Month 4–6）
- SSO/SAML、監査ログ、SOC2 Type I 着手（Cluely 漏洩の対比で「セキュリティが売り」）
- Windows GA、エンタープライズパイロット5社
- モデルルーティング最適化（粗利 80%+ 維持）

### GTM（創業者タスク・コードでは解決しない）
- ローンチ列: X/TikTok デモ動画（Option タップの「魔法の3秒動画」が最強の広告）→ Product Hunt → Show HN → 日本のXテック圏
- 週1回の build in public

## 4. テスト戦略

### 現状マップ
| レイヤー | 現状 | 評価 |
|---|---|---|
| キャプチャ回帰（fixture） | ◎ 509件 + 運用設計 | 業界水準以上 |
| main/IPC ユニット | ◎ モック機構完備 | 良 |
| renderer（React） | ゼロ | App.tsx/各View が無防備 |
| E2E | ゼロ | 縦串（起動→オンボ→生成→ペースト）なし |
| LLM 出力品質 | ゼロ | プロンプト回帰を検知できない |
| 性能（レイテンシ） | ゼロ | 商品そのものが未計測 |
| macOS 上の CI | ゼロ（ubuntu のみ） | swift ヘルパーのコンパイル未検証 |
| バックエンド | 対象がまだ無い | Phase 0 で契約テスト必須 |

### 増設（優先順）
1. **LLM 評価ハーネス**: 既存 context fixture を入力に実プロンプト→実モデル生成、LLM-judge 採点
   （言語一致 / 前置きなし / 画面適合 / SNSで会社メモ混入なし / 長さ）。夜間CIで予算上限付き、プロンプト変更PRの回帰ゲート。
2. **E2E（Playwright + Electron、macos-14 ランナー）**: 起動→オンボ→設定→モックLLM生成→クリップボードまで。
   実ペーストは AX 権限が CI 付与不能のためローカル手動チェックリスト。
3. **macOS CI ジョブ**: unit + `swiftc -parse` + `electron-builder dir` パッケージスモーク（起動→終了）。
4. **renderer テスト**: vitest + React Testing Library。App.tsx の状態遷移を優先。
5. **性能ベンチ**: captureTrace のステージ計時ベース、P50/P95 ソフト予算アサーション。
6. **バックエンド契約テスト**: 認証・クォータ・SSE・Stripe webhook 冪等性。
7. **リリーステスト**: ステージング更新チャネルの自動更新実地手順書。
8. **セキュリティ**: secret-scan CI、pnpm audit、IPC 境界テスト。

## 6. リスク登録簿

| リスク | 影響 | 対策 |
|---|---|---|
| macOS 権限UXの脱落（AX+画面収録の2関門） | activation 半減 | オンボ動画埋め込み、権限なし縮退モード、離脱点計測 |
| 推論コストで粗利崩壊 | $20 が赤字化 | tap=小型モデル既定、コンテキスト予算化、キャッシュ、使用量上限 |
| ペースト信頼性（アプリごとの挙動差） | 「魔法」が壊れる | アプリ別E2Eチェックリスト、失敗時クリップボード保持+通知 |
| 画面データの信頼事故（Cluely 型） | 致命傷 | ローカルファースト維持、リダクション既定ONの検討、送信データ可視化UI |
| OS純正（Recall/Cowork）との衝突 | 長期存続 | 「会社メモ×書き味」の垂直特化、チームデータのネットワーク効果を急ぐ |

## 7. アーキテクチャ決定（Phase 0 の技術方針）

- **バックエンド**: Hono + Cloudflare Workers（低運用コスト・グローバルエッジ・SSE対応）。代替: Fly.io。
  責務は認証 / 従量メーター / モデルルーティング / SSE 中継のみに限定。画面データの保存はしない（ローカルファースト維持）。
- **認証**: Clerk または Supabase Auth（メール+OAuth）。アプリ側はデバイストークン + アカウントJWT。
- **課金**: Stripe Checkout + Customer Portal + webhook（冪等）。エンタイトルメントはバックエンドで判定しアプリにキャッシュ。
- **モデル**: tap 経路 = Haiku級高速モデル既定、パネル/アクション = Sonnet級。BYOK はフォールバックとして温存。
- **テレメトリ**: PostHog（EU/US選択、匿名ID、設定でオプトアウト、画面テキストは絶対に送らない）+ Sentry。
